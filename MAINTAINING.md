# Maintaining the Retrac Labs Website

This site is a React + Vite app. If you are used to plain HTML/CSS/JS, the main shift is this:

- HTML is mostly generated by React components.
- Reusable page sections live in `src/app/components/`.
- Project text/content lives in `src/app/data/projects.ts`.
- Styling is mostly Tailwind classes written directly in `.tsx` files.
- The local dev server is Vite, started with `npm run dev`.

## Daily Workflow

1. Open the project in WebStorm:

   ```text
   ~/WebstormProjects/retraclabs.github.io
   ```

2. Start the dev server:

   ```bash
   npm run dev
   ```

3. Open the local URL Vite prints, usually:

   ```text
   http://localhost:5173/
   ```

4. Edit files in WebStorm. Vite should refresh the browser automatically.

5. Before committing, run:

   ```bash
   npm run build
   ```

6. Commit and push in GitHub Desktop.

### `index.html`

The browser entry shell. You rarely edit this. It contains:

```html
<div id="root"></div>
<script type="module" src="/src/main.tsx"></script>
```

### `src/main.tsx`

Starts React and loads the app:

```tsx
import App from './app/App';
import './styles/index.css';
```

You rarely edit this.

### `src/app/App.tsx`

This is the main traffic controller. It decides whether to show:

- the homepage
- an individual project page
- the privacy policy page

Current route patterns:

```text
#/
#apps
#about
#/privacy
#/projects/amparo
#/projects/snippystack
#/projects/nexus
```

If a page does not show when clicked, check `App.tsx` first.

## Editing Homepage Sections

### Hero / Big Opening Animation

Edit:

```text
src/app/components/Hero.tsx
```

This controls:

- the large RETRAC / LABS text
- the scroll animation
- the small badge near the top
- the “Scroll to Dive In” hint

To change the badge text, find:

```tsx
APPS IN DEVELOPMENT
```

To remove the badge entirely, remove the `motion.div` block containing `Sparkles`.

### Lab / Project Grid

Edit:

```text
src/app/components/LabSection.tsx
```

This controls:

- intro copy under the hero
- the “Current Projects” heading
- the clickable project cards
- which project each card opens

Project card buttons call:

```tsx
openProject('amparo')
```

That must match a project `slug` in `src/app/data/projects.ts`.

## Editing Project Information

Edit:

```text
src/app/data/projects.ts
```

Each project is an object:

```ts
{
  slug: 'amparo',
  name: 'Amparo',
  platform: 'iOS',
  status: 'Beta Testing',
  accent: 'pink',
  accentText: 'text-pink-400',
  icon: Smartphone,
  summary: 'A short tagline.',
  description: 'A longer paragraph.',
  highlights: [
    'Bullet one',
    'Bullet two',
  ],
  nextSteps: [
    'Next step one',
    'Next step two',
  ],
}
```

### What Each Field Does

- `slug`: URL identifier. Example: `#/projects/amparo`.
- `name`: project title.
- `platform`: small platform pill.
- `status`: status pill.
- `accent`: used for glow color logic in `ProjectDetail.tsx`.
- `accentText`: Tailwind text color class.
- `icon`: icon from `lucide-react`.
- `summary`: large intro text on cards and project page.
- `description`: main body text on the project page.
- `highlights`: bullets in “What It Is.”
- `nextSteps`: bullets in “Next Steps.”

## Adding A New Project Page

1. Open:

   ```text
   src/app/data/projects.ts
   ```

2. Import an icon if needed:

   ```ts
   import { Monitor, Smartphone, Zap, Shield, type LucideIcon } from 'lucide-react';
   ```

3. Add a new project object to the `projects` array:

   ```ts
   {
     slug: 'new-app',
     name: 'New App',
     platform: 'iOS',
     status: 'Concept',
     accent: 'cyan',
     accentText: 'text-cyan-400',
     icon: Shield,
     summary: 'Short public-facing tagline.',
     description: 'Longer explanation of what this app is and why it exists.',
     highlights: [
       'Main feature one',
       'Main feature two',
       'Main feature three',
     ],
     nextSteps: [
       'Design prototype',
       'Start beta testing',
       'Prepare App Store materials',
     ],
   }
   ```

4. Open:

   ```text
   src/app/components/LabSection.tsx
   ```

5. Add or duplicate a project card and make sure it calls:

   ```tsx
   openProject('new-app')
   ```

6. Run:

   ```bash
   npm run build
   ```

## Editing Individual Project Page Layout

Edit:

```text
src/app/components/ProjectDetail.tsx
```

This controls the shared layout for every project page.

If you want every project page to show a new section, add it here.

Example section:

```tsx
<section className="border-4 border-zinc-800 bg-zinc-900 rounded-[1.5rem] sm:rounded-[2rem] p-6 sm:p-8 mt-6">
  <h2 className="text-2xl font-black text-white mb-4">Privacy</h2>
  <p className="text-zinc-300 font-medium leading-relaxed">
    Privacy copy goes here.
  </p>
</section>
```

If you want each project to have different custom sections, add more fields to the project data first, then render them in `ProjectDetail.tsx`.

## Adding Images

Best simple approach: put images inside:

```text
src/app/assets/
```

Create that folder if it does not exist.

Example:

```text
src/app/assets/amparo-screen.png
```

Then import it inside a component:

```tsx
import amparoScreen from '../assets/amparo-screen.png';
```

If the component is deeper, the path may need another `../`.

Example usage:

```tsx
<img
  src={amparoScreen}
  alt="Amparo cycle overview screen"
  className="w-full rounded-2xl border-2 border-zinc-800"
/>
```

### Where To Add Images For Project Pages

Use:

```text
src/app/components/ProjectDetail.tsx
```

Or add an image field to each project in `projects.ts`:

```ts
import amparoScreen from '../assets/amparo-screen.png';

{
  slug: 'amparo',
  image: amparoScreen,
}
```

Then update the `Project` type:

```ts
image?: string;
```

And render conditionally:

```tsx
{project.image && (
  <img
    src={project.image}
    alt={project.name + ' screenshot'}
    className="w-full rounded-2xl border-2 border-zinc-800"
  />
)}
```

## Styling Basics

Most styling is Tailwind CSS inside `className`.

Examples:

```tsx
className="text-white bg-zinc-900 p-6 rounded-2xl"
```

Common patterns:

- `text-white`: white text
- `text-zinc-400`: gray text
- `bg-zinc-900`: dark background
- `p-6`: padding
- `mt-6`: margin top
- `rounded-2xl`: rounded corners
- `border-2 border-zinc-800`: border
- `sm:`, `md:`, `lg:`: responsive breakpoints

Example:

```tsx
className="text-lg sm:text-xl md:text-3xl"
```

This means:

- mobile: `text-lg`
- small screens and up: `text-xl`
- medium screens and up: `text-3xl`

## Deployment

The deployment workflow lives here:

```text
.github/workflows/deploy.yml
```

On push to `main`, GitHub Actions should:

1. Install dependencies with `npm ci`
2. Build with `npm run build`
3. Upload `dist/`
4. Deploy to GitHub Pages

If Actions does not run:

1. Confirm the workflow file exists in GitHub.
2. Go to GitHub repo settings.
3. Open Pages settings.
4. Set Pages source to GitHub Actions.
5. Push a new commit.

## Common Problems

### The project page does not open

Check:

1. The card calls the correct slug in `LabSection.tsx`.
2. The slug exists in `projects.ts`.
3. `App.tsx` route matching still uses:

   ```ts
   /^#\/projects\/([a-z0-9-]+)$/
   ```

### The site is blank

Run:

```bash
npm run build
```

Read the first error. It is usually:

- a bad import path
- mismatched JSX tags
- a missing comma in `projects.ts`
- a missing icon import

### Text updates do not show

Hard refresh the browser. If needed, stop and restart:

```bash
npm run dev
```

### GitHub Pages does not update

Check:

1. GitHub Desktop pushed the commit.
2. GitHub Actions ran.
3. The deploy workflow completed successfully.
4. Browser cache is not showing the old site.

## Mental Model

Think of the site as three layers:

1. Data: `src/app/data/projects.ts`
2. Components/layout: `src/app/components/*.tsx`
3. Routing: `src/app/App.tsx`

Most routine edits are either:

- copy edits in `projects.ts`
- layout edits in one component
- privacy/footer edits in `PrivacyPolicy.tsx` or `Footer.tsx`

When in doubt, run:

```bash
npm run build
```

It catches most structural mistakes before GitHub does.
